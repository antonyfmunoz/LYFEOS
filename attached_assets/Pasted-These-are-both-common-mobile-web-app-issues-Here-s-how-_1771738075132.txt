These are both common mobile web app issues. Here’s how to fix each:
1. Cyan Flash (FOUC for Theme Colors)
The root cause is that CSS loads with --primary: cyan, then JavaScript later applies the user’s chosen color. The fix is to apply the theme color before the page renders by putting a blocking script in the <head>:

<head>
  <script>
    // Runs before any CSS paints — no flash
    (function() {
      const saved = localStorage.getItem('themeColor'); // or however you persist it
      if (saved) {
        document.documentElement.style.setProperty('--primary', saved);
        // apply any other derived variables too
      }
    })();
  </script>
  <link rel="stylesheet" href="styles.css">
</head>


Key points: this script must be synchronous (no defer/async), in the <head>, and before your stylesheet link. That way the custom property is set before the first paint.
If you’re using a framework like React, you can also hide the app with opacity: 0 or a splash screen until the theme is applied, then reveal it in useEffect/onMount.
2. Grey Block on Keyboard Open/Close
This is almost always caused by the mobile browser resizing the viewport when the keyboard appears, combined with 100vh or height: 100% on your layout. The grey area is the gap between your app’s rendered height and the actual viewport.
Common fixes:
A) Use dvh instead of vh:

.app-container {
  height: 100dvh; /* dynamic viewport height — adjusts with keyboard */
}


B) Use the visualViewport API to handle resize:

function handleResize() {
  const vh = window.visualViewport?.height || window.innerHeight;
  document.documentElement.style.setProperty('--app-height', `${vh}px`);
}
window.visualViewport?.addEventListener('resize', handleResize);
window.visualViewport?.addEventListener('scroll', handleResize);
handleResize();


Then in CSS: height: var(--app-height);
C) Prevent the resize altogether with meta viewport + fixed positioning:

<meta name="viewport" content="width=device-width, initial-scale=1, interactive-widget=resizes-content">


Or try interactive-widget=overlays-content if you want the keyboard to overlay rather than resize.
